Regular Expressions

Monday, February 22, 2016

In all of our previous exercises, when we had to search for patterns within the information given, we had to create our loops and substitution methods manually. This is a great way to learn the fundamental concepts of programming, since we had to basically build everything from scratch. But is there an easier way? Of course there is. We can use regular expressions to quickly search a large amount of information for patterns and work with that data.

A regular expression is an expression that can be used to match a pattern against strings. They can be used to find strings that contain the matching expression as a substring, or even extract the matching expression from the original string. Thus, in the above comic, the hero can use a regular expression to search through a huge database for any substring matching the standard format of an email address.

Regular Expressions in Ruby

A regular expression is defined between forward slashes, like so:

/expression/

By itself, it doesn't do much. However, you can use the match operator '=~' to determine if the regular expression matches with a larger string.

/expression/ =~ "I contain the expression"
# => 14

The match operator returns the first index in the larger string where the match is found - in this case, the expression starts at the 14th index. If no match is found, the match operator instead returns nil.

There are a lot of tricks you can do with regular expressions to make them more useful and flexible. Here are some examples:

Searching for patterns with wild cards within a range:
/w[aeiou]ld/ => searches for any of the following patterns: wald, weld, wild, wold, wuld
/1[0-9]/ => searches for any pattern within the range 10 to 19

/^f/ => reverses the pattern, in this case, searching for any string WITHOUT 'f'
Regular Expressions in JavaScript

Regular expressions in JavaScript behave the same way as you would expect in Ruby. They are also used as an assistive tool to search and replace text in a large amount of information. The syntax is slightly different for JavaScript:

var pattern = /expression/modifier

As this is JavaScript, we need to define our expression as a variable by declaring it with the var keyword. The expression itself is within the forward slashes, just as we would delineate it with Ruby. Directly following the second forward slash is a modifier that can optionally filter the results. Modifiers can specify the search to be case-sensitive or insensitive, finding multiple matches globally, or find matches across multiple lines.

The syntax for adapting the search for wild cards and multiple values is similar to Ruby:

[abc] => Find any character within the brackets
[^abc] => Find any character NOT within the brackets
[0-9] => Find any digit within the range
[x|y] => Find y as an alternative to x

This is just a basic overview of what regular expressions can do. As we begin working with larger sets of data within more complex programs, regular expressions will become invaluable for searching through data quickly and efficiently. There are many more modifiers that can be added to regular expressions that can make them very powerful and flexible tools.
