Creating Your Own Class

Saturday, February 6, 2016

When I'm not busy with learning the ins and outs of web development, I like to prototype and design new board games as a hobby. Designing a board game can be a very time-consuming process, especially if the game has a lot of different cards or components that need to be created from scratch. One way to save time when creating a large number of different cards is to create a base template that can be easily modified. For instance, if you know that all your cards are going to follow the same format, you can create a template of that format and fill in the blanks with specific information, thus preventing a lot of repeated work by drawing out every card one at a time.

Classes in Ruby work in a very similar fashion. You can think of a class as a template that can generate a certain type of object. Every object generated by that class will share the same set of properties. However, you can modify those properties for each object so that each object ends up being unique. Let's say that you were trying to build a class as a template for generating a deck of cards. Each card follows the same format - a number, a suit, and some graphical image. But each card is also unique - there are no repeated combinations of suits and numbers. In this case, the number, suit, and graphic are all common properties contained in the card class. However, the specific values for each card can be modified by creating class and instance variables.

class Card

	def initialize
		@name = "Ace"
		@value = 1
		@suit = "Hearts"
	end

end

What we have here is the bare minimum we need to create a class. We've defined a class called Card, and within it we've included a class method called initialize. This method allows us to call a special method .new to create a new object that belongs to the card class. When the initialize method is called, it generates a new object with three variables. These variables have the "@" symbol in front of them, which makes them instance variables. Instance variables are different than local variables because they can be used within other class methods, allowing you to pass in extra information into a method and avoiding excessive local variables. It also means that any object, or instance, generated from this class will have these variables defined.

Of course, having a deck of cards containing only Aces of Hearts will make for some pretty boring poker games. Just like any method, we can add arguments to the initialize method to assign different values to the same instance variables.

class Card

	def initialize(name, value, suit)
		@name = name
		@value = value
		@suit = suit
	end

end

first_card = Card.new("Ace", 1, "Hearts")

Now we can create variables outside of the class that will share common properties, which we can define when creating the object. This will save us some time when we need to generate an entire deck. We can even assign the cards different numerical values depending on how the cards will be used - the value of an Ace can be changed to 1 or 13 depending on whether aces are high or low.

What else can we do with our methods? Well, we can continue to add new class methods to give these objects unique behaviors. Let's add a simple class method to output a string of what card you're holding.

class Card

	def initialize(name, value, suit)
		@name = name
		@value = value
		@suit = suit
	end

	def display
		puts "This is the #{@name} of #{@suit}!"
	end

end

first_card = Card.new("Ace", 1, "Hearts")
first_card.display #=> "This is the Ace of Hearts!"

Now we've created a display method that can be called on any instance of Card to output some of our instance variables. Notice how we didn't need to pass any arguments into our display method? This is because the instance variables defined when we initialized the object can be carried over to any method within the class.

Classes are a very powerful way to generate many objects that share properties in only a few lines of code! By writing out a model of our object within a class, we can use the class as a blueprint to quickly create as many copies of these objects as we need.
